# Report Draft — AI-enabled Detection and Classification of Exoplanet-like Dips in Noisy TESS Light Curves

## Methodology

We implemented a modular, physics-informed AI pipeline for detecting and classifying transit-like events in TESS light curves. The pipeline is separated into six stages. First, raw TESS light curves are ingested from FITS products or local arrays, preserving SAP flux, PDCSAP flux, centroid columns, quality flags, CROWDSAP, FLFRCSAP, and target metadata. The preprocessing stage removes invalid cadences, applies configurable quality masking, selects PDCSAP or SAP flux with explicit provenance, normalizes the flux, performs conservative detrending, estimates noise, records cadence/gap statistics, and generates preprocessing diagnostics.

Second, periodic dips are detected using a baseline transit-search module. The detector estimates candidate period, epoch, duration, depth, signal-to-noise ratio, number of observed transits, and a periodogram score. Detection is intentionally separated from classification: this stage only asks whether a repeated dip exists.

Third, candidate transit parameters are refined using a robust box-profile model around the detected ephemeris. The pipeline estimates period, epoch, duration, transit depth, depth uncertainty, radius ratio proxy, event-by-event depth scatter, and transit-count reliability. Planet radius in Earth radii is only reported when a stellar radius is available; otherwise the pipeline reports only radius ratio.

Fourth, scientific vetting features are extracted. These include odd/even depth mismatch, secondary eclipse significance at phase 0.5, centroid shift significance, CROWDSAP-based crowding risk, dilution-corrected depth, transit-shape score, red-noise proxy, and data-quality score. These features are designed to distinguish planetary transits from eclipsing binaries, blended/background events, stellar variability, and instrumental/systematic artifacts.

Fifth, we implemented a transparent rule-based baseline classifier. This classifier combines physical evidence into interpretable scores for planetary transit, eclipsing binary, blend/contamination, stellar variability, and systematics. It is used both as a baseline and as a scientific guardrail for the AI classifier.

Sixth, we added a supervised AI classifier trained on a curated labeled candidate catalog. The classifier consumes the feature table generated by the previous stages, normalizes labels into canonical classes, imputes missing values, trains a class-balanced random forest by default, optionally calibrates probabilities when enough examples exist, evaluates using stratified holdout validation, and outputs class probabilities. Final predictions blend AI probabilities with rule-based scientific scores and apply physical guardrails for strong secondary eclipses, odd/even mismatches, centroid shifts, low data quality, and low SNR.

## Classes

The pipeline supports the following classes:

- `PLANETARY_TRANSIT_CANDIDATE`
- `ECLIPSING_BINARY`
- `BLEND_OR_CONTAMINATED_SIGNAL`
- `STELLAR_VARIABILITY`
- `INSTRUMENTAL_OR_LOW_QUALITY_SYSTEMATIC`
- `NO_SIGNIFICANT_SIGNAL`
- `UNCERTAIN_TRANSIT_LIKE_SIGNAL`

This multi-class framing matches the problem statement: periodic dips may be real exoplanet transits, eclipsing stellar companions, blended background sources, starspots, stellar variability, or detector/systematic artifacts.

## Uncertainty and confidence

Signal-to-noise ratio is estimated from transit depth and local out-of-transit noise, scaled by the number of in-transit points. Depth uncertainty is estimated from robust local scatter and event-by-event depth variation. Period and duration uncertainties are first-pass uncertainties from the period-search resolution and box-profile refinement. Classification confidence is the final class probability after supervised prediction, rule-based blending, and physical guardrails. This confidence should be interpreted as pipeline confidence, not formal astronomical validation probability.

## Evaluation plan

For detection and parameter estimation, the pipeline should be validated on synthetic injection-recovery experiments and known TESS targets. Key metrics are recovery rate, false alarm rate, period error, duration error, and depth error. For classification, the curated labeled dataset should be split using stratified holdout validation or cross-validation. Metrics should include accuracy, balanced accuracy, macro F1, class-wise precision/recall/F1, and a confusion matrix. Macro F1 is emphasized because exoplanet datasets are usually imbalanced.

## Visualization

For each candidate, the pipeline can generate preprocessing, detection, and vetting plots. The Part 6 classifier additionally outputs confusion-matrix plots, feature-importance plots, probability summaries, model metrics, and a model card. These visualizations are intended to make the pipeline transparent and evaluator-friendly.

## Assumptions and limitations

The pipeline assumes that transit-like events are approximately periodic within the observed baseline and that detrending does not fully remove the astrophysical signal. CROWDSAP is treated as contamination evidence rather than an automatic rejection. Planet radius is only estimated when stellar radius is available. The supervised classifier depends on the quality and label taxonomy of the curated dataset. Crowded-field candidates require additional high-confidence vetting using target pixel files, Gaia neighbor analysis, or difference imaging before being considered scientifically validated planets.
